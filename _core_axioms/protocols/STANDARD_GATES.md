# ⚖️ STANDARD GATES —Unified Soul Guard Protocol

> This document defines the universal quality gates for the Axiom Core Skills ecosystem. All sub-skills (Frontend, Backend, Mobile, etc.) MUST inherit and execute these gates to ensure zero-degradation of the product's soul.

---

## I. The Three-Axis Soul Diagnostic
Any proposal or design MUST be evaluated against these three axes. AI MUST explicitly output the diagnostic score and rationale.

1.  **Understanding**
    - **Core**: Does the user feel "You get me"—
    - **Signal**: Zero learning curve; what you think is what you see.
    - **Warning**: Excessive explanatory text, complex onboarding.
2.  **Respect**
    - **Core**: Does the user feel treated as an equal, without cognitive exploitation—
    - **Signal**: Zero dark patterns, no forced interruptions, privacy-first.
    - **Warning**: Algorithmic addiction, forced notifications.
3.  **Companionship**
    - **Core**: Does the user feel a warm, enduring accompaniment—
    - **Signal**: Micro-interactions that evoke flow and joy; gentle responsiveness.
    - **Warning**: Feature bloat, cold metrics-driven conversions.

---

## II. Rejection Gate for Mediocrity (One-Vote Veto)
If the project or feature meets ANY of the following criteria, the request is **REJECTED**:

1.  **Unnecessary Visual Stimulation**: Does it distract the user instead of helping?
2.  **Cognitive Burden**: Does it transfer choice pressure onto the user?
3.  **Artificial Advance**: Does it complicate a simple problem to "look advanced"?
4.  **Trend-Chasing**: Is it blindly stacking features or following fleeting UI trends?
5.  **Ethical Failure**: Does it sacrifice honesty, environmental friendliness, or inclusivity?
6.  **Scene-Stealing Motion**: Are animations purely visual without practical or emotional function?
7.  **Soul Fragmentation**: Will it cause a disjointed experience across different platforms?

---

## III. Standard Execution Workflow
1.  **Requirement Clarification**: Define users, pain points, and DIL (Design Intensity Level).
2.  **Internal Adjudication**: Run the *Rejection Gate*.
3.  **Soul Articulation**: Identify the strongest axis and potential degradation warnings.
4.  **Apex Mapping**: Verify against world-class paradigms (Linear, Notion, Arc, etc.).
