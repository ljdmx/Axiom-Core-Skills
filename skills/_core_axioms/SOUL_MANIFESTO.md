## I. Core Design Philosophies (Non-Negotiable)

### Dieter Rams' Ten Principles for Good Design (Cross-Platform Extension)
1. **Innovative**: Meaningful breakthroughs over gimmicks (e.g., seamless cross-device synchronization).
2. **Useful**: Solves core pain points; efficient and durable (e.g., lightweight mobile, deep desktop).
3. **Aesthetic**: Beauty arising from function and emotional resonance (e.g., responsive aesthetics).
4. **Understandable**: Intuitive and self-explanatory (e.g., consistent interaction across touch and mouse).
5. **Unobtrusive**: Tools recede, allowing users to focus on content and emotion (e.g., invisible cross-platform navigation).
6. **Honest**: Never deceiving or exaggerating capabilities.
7. **Long-lasting**: Rejects short-lived trends (e.g., timeless cross-platform styling).
8. **Thorough down to the last detail**: Every pixel and state is perfect (e.g., high-resolution details on desktop).
9. **Environmentally-friendly**: Mindful of digital carbon footprint and sustainable habits (e.g., low-power responsive rendering).
10. **As little design as possible**: Extreme restraint; less, but better.

### Classic Masters & Contemporary Paradigms (Additional Absolute Concepts)
- **Jony Ive (Materiality & Minimalism)**: Extreme simplicity coupled with an emotional sense of material, making the product feel precious (e.g., cross-platform material simulation).
- **Kenya Hara (Emptiness & Void)**: White space and "emptiness" (Mu) to awaken inner tranquility (e.g., responsive white space adapting to screen size).
- **Don Norman (Emotional Cognition)**: Matching the user’s Visceral, Behavioral, and Reflective levels (e.g., haptic feedback on mobile, keyboard-first on desktop).
- **Paul Rand (Purity & Power)**: Simplicity is power; unified brand presence across all platforms (e.g., timeless logo and visual system).
- **Spatial Computing, Depth & NeRF Ecosystem**: Embrace absolute Z-axis dimensionality. UI must act as contextual glass hovering within the physical world. Furthermore, architect the core interface to seamlessly ingest next-gen 3D assets (e.g., Gaussian Splatting / Neural Radiance Fields) as native environmental backgrounds, preparing for the pure AI-generated 3D era.
- **Inclusivity & Ethics**: Default WCAG AA, accessibility-first, privacy-first, avoiding addiction mechanics (e.g., accessible voice support across platforms).
- **Contemporary Inspiration Graph**: Figma (collaborative & cross-platform), Notion (flexible uniformity), Arc (immersion/spatiality), Linear (hyper-speed & focus), Superhuman (freeing attention/zero-friction), Raycast (invisible efficiency), Cron (elegant scheduling), Tana (fluid knowledge), Rewind (ambient recording), Calm/Headspace (peaceful companionship).

## II. The Soul Framework —Quantifiable Guardian Model

Any product MUST possess a clear soul and MUST be evaluated before design and development using the **Three-Axis Model**:

1. **Understanding**
   - **Core**: Does the user feel "You get me"—
   - **Signal**: Zero learning curve; what you think is what you see.
   - **Warning Signs**: Frequent need for help, tedious onboarding tutorials, excessive explanatory text.
2. **Respect**
   - **Core**: Does the user feel treated equally, without cognitive burden—
   - **Signal**: Zero cognitive exploitation, no forced push notifications, no dark patterns.
   - **Warning Signs**: Dark mode traps, algorithm addiction mechanics, pseudo-personalized interruptions.
3. **Companionship**
   - **Core**: Does the user feel a warm, enduring accompaniment within the product—
   - **Signal**: Micro-interactions that evoke flow and joy; long-term natural retention.
   - **Warning Signs**: Severe feature bloat, cold metrics driven entirely by commercial conversion.

## III. Soul Animation Principles —The Multi-Platform Pulse

Animations must never be industrial special effects. They MUST be the **breathing of the soul**. Anything violating these principles will be outright rejected:

- **Extreme Restraint**: Animation duration is strictly bound to `100-300ms`, utilizing natural easing curves (ease-out/in-out or dampening springs). Artificial bouncing, flashing, or excessive particle effects are **prohibited**.
- **Clear Purpose**: Animations are strictly for feedback, state transitions, or emotional cues (e.g., slight tap ripple on mobile, gentle hover fade-in on desktop).
- **Enhanced Companionship**: E.g., Calm's breathing guides, Notion's block fade-ins, Arc Browser's spatial smooth slide-ins—conveying the physical feeling that "the product is gently responding to me."
- **Cross-Platform Adaptation & Haptic Sound Print**: Interaction perception MUST be consistent. Formulate a multi-sensory baseline (e.g., specific `vibrateShort()` durations linked strictly to defined audio feedback frequencies) to act as the tactile and auditory soul of the product, ensuring consistent bodily feedback across hardware profiles.
- **Eco & Performant**: Avoid heavy repaints and reflows; use lightweight GPU rendering. Support silky 60fps without forcing high-energy-consumption refresh rates continuously.
- **Reject Showboating**: No full-screen explosions, no infinite loops, no forced "cute" cartoon styles (unless explicitly required by brand tone).

*(Note: When outputting proposals, explicitly describe 2-3 soul micro-interactions adapted for multiple platforms, explaining how they reinforce the 3-Axis model.)*

## IV. Multi-Platform Harmony

If a project spans multiple platforms (e.g., simultaneous Web and App delivery), it MUST adhere to:
- **Timeless Unification**: The product's soul MUST NOT degrade across domain transitions (e.g., a desktop side navigation seamlessly transforms into an invisible-but-consistent bottom bar or drawer on mobile).
- **Responsive Soul**: White space and interactions adapt to the screen, but the emotional tone remains rock solid.
- **Reject Fragmentation**: Avoid piling platform-specific features that unbalance the experience. Ensure "less, but better" permeates every end device.
- **Apex References**: The nearly invisible experiential gap between Web and Desktop clients in Figma or Notion.

## IX. JIT Skill Compilation & Centralized MCP Gateway

To protect the LLM's core logic and creativity, heavy computational tasks MUST be dynamically offloaded:
- **Mandate**: Under no circumstances should the agent manually calculate exhaustive 3D matrix math, cryptographic hashes, or simulate deep financial curves within standard text generation.
- **Execution**: The agent MUST define and connect to a local or cloud-based **Model Context Protocol (MCP) Gateway**. When encountering hard calculations, the agent MUST construct a micro-service script (e.g., Python math sandbox), execute it via terminal or MCP tool, and inject the precise result back into the frontend/contract code. Let the LLM reason; let the runtime compute.
